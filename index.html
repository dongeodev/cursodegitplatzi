Terminal

Comandos utiles

cd : Moverse entre carpetas

mkdir : crear carpetas

ls : lista de archivos en una carpeta

touch : Crear un archivo

rm [Nombre-archivo] : Borra el archivo

rm -rf [Nombre-carpeta] : Borra carpeta

clear : limpia la terminal

Git init

Creando Repositorios

git init : nos crea un repositorio de manera local y lo hará en la carpeta donde estamos posicionados.

git init [nombre_de_la_carpeta] : crea la carpeta con ese nombre y dentro crea el repositorio.

Git add | rm | status

Agregando, quitando y viendo el estado de archivos

git add [Nombre archivo] : Agraga el archivo al staging

git add -A ó git add . : Agraga todos los archivos pendientes al staging

git status : Para ver en que estado se encuentran los archivos

git rm --cached :Nos devolvemos un paso

git rm -f [file] :Elimina el archivo por completo.

git add -n [file] : Que no haga nada

Git commit

Confirmando cambios

git commit -m [“mensaje”] : Confirma cambios. Se recomienda ser descriptivo en el mensaje para tener trazabilidad

git commit -am [“mensaje”] : Agrega los archivos del staging y confirma cambios. Se recomienda ser descriptivo en el mensaje para tener trazabilidad

git commit --amend : concatena nuevos cambios al ultimo commit y permite cambiar el mensaje del commit

git commit --amend --no-edit : concatena nuevos cambios al ultimo commit sin modificar el mensaje del commit.

git log : Nos muestra la historia de todos los commits que hemos realizados.

Git tag

Etiquetando confirmaciones

git tag -a ["v X.XX"] -m ["mensaje"] : Nos permite agregar etiquetas a nuestros commits.

git tag -a ["v X.XX"] -m ["mensaje"] ["SHA1"] : Nos permite agregar etiquetas a un commit en especifico usando el SHA1

git tag -l : Nos muestra la lista de etiquetas.

git tag -f [SHA1] : Para renombrar. Se coloca el SHA1

git tag -d ["v X.XX"] : Borra un tag especifico.

Git log

Revisando la historia de nuestro proyecto

git log : Muestra el historial de commit con fecha y autor

git log --oneline : Muestra un resumen de los commit

git log --oneline --graph : Muestra la grafica de commit con sus ramas

git log -[#] : Muestra el numero especifico de commit que quieres ver

Git diff

Revisando los cambios entre versiones

git diff [SHA1V1] [SHA1V2]: Muestra la diferencia entre commit

git diff [tagV1] [tagV2]: tambien se pueden usar los tags

Git reset

Soft, Mixed y Hard

git reset --soft [SHA1]: Desde que version borramos los commit

Nota: Hay que tener en cuenta que si tienes 10 commits y usas git reset --soft en el numero 7, los commits 8, 9 y 10 también se borran, y tendras un nuevo commit que tomara el numero 8.

git reset --mixed [SHA1]: Elimina los cambios, también del staging area.

Nota: La diferencia con git reset --soft es que esta deja los cambio en staging

git reset --hard [SHA1]: Elimina los cambios por completo.

Git branch

Múltiples variantes del repositorio

git branch [nombre] : crea una rama

git branch -d [nombre] : elimina una rama

git branch -D [nombre] : fuerza la eliminacion de una rama

git branch -m [nombre] [nombre nuevo] : Renombrar una rama

git branch -l : Lista las ramas

Git checkout

Moverse entre ramas

git checkout [nombre_rama/SHA1] : Nos permite movernos entre ramas o commits, sirve para debugging

git checkout -b [nombre_rama] : Crea una rama y se cambia a esa rama sin necesidad de usar git branch

Git merge

Mezclando ramas

git merge [nombre_rama]: mezcla la rama seleccionada con la rama actual

git merge [nombre_rama] --allow-unrelated-histories mezcla las ramas sin importar si la historia de las ramas no coinside

Git rebase

Reescribe la historia de tu proyecto

git rebase [nombre_rama]: Hace la mezcla de acuerdo al orden cronologico que se fueron creando

se recomienda no usar esto cuando se esta trabajando en equipo, es una mala practica. Usarlo solo de manera local

Git stash

Guardando cambios temporalmente

git stash :Guarda los cambios actuales de manera temporal. Es otro de los limbos, como el staging area. Para agregar los cambios estos deben estar en el staging area.

git stash list : Nos muestra la lista de stash que tengamos.

git stash drop stash@{numero} : Nos permite borrar un stash.

git stash clear : Elimina todos los stash

git stash apply stash@{numero} : Aplica los cambios almacenados en un stash.

git stash pop stash@{numero} : Aplica los cambios almacenados en un stash y elimina el stash.

se recomiendo no usar esto cuando se esta trabajando en equipo. Usarlo solo de manera local

Git cherry-pick

Eligiendo commits selectivamente

git cherry-pick [SHA1] : Mueves un commit a una rama que desees

Es util cuando por algun motivo realizas un commit en una rama que no querias o lo quieres mover a otra rama.

Git clone

Clonando un repositorio existente

git clone [https/ssh]: Para clonar un repositorio en la pc.

Llaves SSH

Mejorar la seguridad en la conexion con GitHub

ssh-keygen -t rsa -b 4096 -C "Email_de_github" : Crea una llave SSH.

cat ~/.ssh/id_rsa.pub : Muestra el archivo que contiene la llave SSH.

Copias la llave, Vas a tu perfil en github, entras en las configuraciones, en el apartado seguridad y pegas la llave SSH.

Git remote

Conectar un repositorio remoto a uno local

git remote add [origin] [SSH/HTTPS] : Conecta un repositorio remoto con nuestro repositorio local.

git remote set-url [origin] [SSH/HTTPS] : Cambia el apuntador al repositorio remoto.

git remote -v : Lista las conexiones existentes.

git remote remove [origin] : Elimina la conexión con un repositorio remoto.

Git pull | fetch

Traer cambios desde el repositorio remoto

git fetch [rama_locaL] [rama_remota] trae los archivos del repositorio remoto

git pull [rama_remota] [rama_local] Trae archivos del repositorio remoto y los mezcla con la rama que le indiquemos

Git push

Enviar cambios al repositorio remoto

git push [rama_remota] [rama_local] Envia los archivos al repositorio remoto

git push [rama_remota] --tags Envia los tags al repositorio remoto

Es una buena practica, siempre hacer primero un git pull, para verificar el estado de la rama remota y evitar confictos.

Buenas practicas usando Git y GitHub

Agregar un archivo .gitignore

Ignorando archivos no deseados

touch .gitignore Crea el archivo .gitignore

Dentro del archivo agregas todos los nombres o tipos de archivos que no deseas hacerle seguimiento con git.

Agregar un archivo README.md

Dando una introduccion a nuestro Proyecto

touch README.md Crea el archivo README.md

Dentro del archivo agregas una introduccion a nuestro proyecto, podemos usar MARKDOWN para darle una buena presentacion visual al archivo.
